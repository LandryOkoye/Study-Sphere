# StudySphere — Your AI-Powered Study Coach Agent

> **StudySphere is not just a learning platform. It is an autonomous AI agent that plans, teaches, quizzes, and adapts — so students can stop guessing what to study and start making real progress toward their exam goals.**

---

##  The Problem We Solve

Students preparing for high-stakes exams (WAEC, JAMB, university finals, and more) face three compounding problems:

1. **No clear plan.** Most students open a textbook without knowing which topic to tackle today, how much time to spend, or what to prioritise given their weaknesses.
2. **No personalised feedback loop.** Generic study apps present the same content to every student regardless of what they already know, making revision inefficient and demoralising.
3. **No continuity.** Progress notes, quiz results, and study materials are scattered across PDFs, notebooks, and cloud drives with no single system tracking growth over time.

These problems don't stem from a lack of effort — they stem from a lack of an intelligent system that can *think on behalf of the student*.

**StudySphere is that system.**

---

##  StudySphere Is an AI Agent — Here's What That Means

At the heart of StudySphere is a **Study Coach Agent** — a fully autonomous AI that doesn't just answer questions, it *acts*:

| What a typical study app does | What the StudySphere Agent does |
|---|---|
| Shows you content | **Creates a personalised day-by-day study plan** based on your goal, timeline, and confidence level |
| Gives you practice questions | **Generates contextual quizzes on exactly the topic you just studied** |
| Scores your answers | **Grades your quiz with detailed reasoning** powered by 0G Compute (Qwen2.5) |
| Shows the same material to everyone | **Identifies your weak topics** from quiz performance and re-schedules them automatically |
| Stores files on a central server | **Persists every lesson, quiz, and reflection to 0G decentralised storage**, returning verifiable root hashes |

The agent runs a continuous loop:

```
Set Goal → Generate Plan → Choose Next Action → Generate Lesson or Quiz
   ↑                                                        ↓
Update Weak Topics ←──── Grade Answers ←──── Student Submits Answers
```

Every step of this loop is executed by the backend orchestrator (`src/lib/agent/studyCoach.ts`) via **0G Compute**, with all generated artefacts permanently stored on **0G Storage**. Agent memory is persisted between sessions so the student picks up exactly where they left off.

### Study Coach Agent Capabilities

- 🗺 **Autonomous planning** — Given a subject, exam type, duration, and target score, the agent generates a structured multi-day study plan
-  **Lesson generation** — Creates full written lessons on the recommended topic for that session
-  **Quiz generation** — Produces tailored quiz questions immediately after a lesson
-  **Automatic grading** — Sends student answers to 0G Compute, which scores and explains each answer
-  **Adaptive re-prioritisation** — Tracks weak topics and surfaces them in future sessions
- **Decentralised artefact storage** — Every plan, lesson, quiz, and reflection is uploaded to 0G Storage and its root hash is saved in the agent's state

##  What Makes StudySphere Different?

While most learning platforms rely on centralised servers and static content, StudySphere combines an autonomous AI agent with a decentralised infrastructure.

### 1. AI Agent — Not Just a Chatbot
The Study Coach Agent is a *goal-driven agent*: it decides what action to take next (generate lesson, generate quiz, recommend review) based on the current state of your plan and past quiz performance. It does not wait to be told what to do.

### 2. Decentralised Storage (0G Storage)
All educational content and agent-generated artefacts are stored on **0G Storage** — censorship-resistant, permanently accessible, and verifiable by root hash. No single point of failure. No platform lock-in.

### 3. Powerful AI Inference (0G Compute & Qwen2.5)
Every agent decision, lesson, quiz, and grading response is generated by **Qwen2.5** running on **0G Compute** — a decentralised inference network that keeps AI computation off centralised cloud providers.

### 4. Integrated Study IDE
The Study IDE gives students a focused reading environment with a curriculum explorer, AI chat assistant, and a direct link to the Study Coach Agent — all in one resizable split-pane interface.

### 5. Custom Study Libraries
Users can curate topic collections into shareable "Libraries" — distraction-free study views they can return to or share with peers.

##  How to Use the Study Coach Agent

1. **Open a subject** from your dashboard and enter the Study IDE.
2. Click the **Study Coach** button in the IDE toolbar — this navigates to the dedicated agent page (`/agent?subject=<id>`).
3. **Set your goal** — enter your exam type, duration, daily study time, confidence level, and target score. The agent will remember this.
4. Click **Run Next Step**. The agent will decide what action to take (generate a plan on the first run, then lessons and quizzes on subsequent runs).
5. **Answer the quiz** when one is generated, then click **Grade Answers**. The agent grades your responses and updates your weak-topic list automatically.
6. Every artefact the agent creates is uploaded to **0G Storage** — you can see the root hashes in the *Stored Artifacts* panel.

---

##  How to Use the Study IDE

1. **Accessing the IDE**: Click any course or textbook from your dashboard.
2. **Curriculum Explorer**: Use the left panel to browse chapters and topics. Clicking a topic generates a lesson via 0G Compute.
3. **AI Chat**: The right panel gives you a live AI chat assistant for on-the-spot questions.
4. **Study Coach**: Click the *Study Coach* button in the toolbar to open the full-page agent view.
5. **Creating a Library**: Click "Library" in the IDE toolbar to build a curated, shareable topic collection.

##  Getting Started

### Prerequisites
- Node.js (v16 or higher)
- npm, yarn, pnpm, or bun
- Valid environment variables for Web3Auth, Interswitch, and 0G services

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/Chiamaka-odike/Stud.git
   cd Stud
   ```

2. **Install dependencies:**
   ```bash
   npm install
   # or yarn install / pnpm install / bun install
   ```

3. **Set up environment variables:**
   Create a `.env.local` file in the root directory and add the necessary API keys for Web3Auth, Interswitch, and 0G SDKs based on `.env.example`.

4. **Run the development server:**
   ```bash
   npm run dev
   ```

5. Open [http://localhost:3000](http://localhost:3000) in your browser.
